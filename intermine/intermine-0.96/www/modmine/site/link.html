Linking to modMine

Links to report pages

Links can be set up directly to the report pages of many types of entry in modMine. A link isn't to a specific page but runs a search for the given identifier. This searches all identifiers, symbols and synonyms in modMine and will bring up either a report page or a list of possibilities. This is exactly the same as running a Quick Search from the modMine home page.

The URL to use takes the form: intermine.modencode.org/query/portal.do?externalid=identifier&class=identifierClass&origin=MySite

Note that using intermine.modencode.org/query will always redirect to the latest modMine release, please don't create links to the URL of a particular release.

The origin in the URL is optional but will allow us to show a specific welcome/help message to incoming users from your site. It also helps us keep track of incoming links. Let us know what you want to use.

As an option, you can specify an organism name by using organism in the URL, this is useful when linking with symbols that are used acros organisms. The organism name must be the shortName as defined in modMine, for instance D.%20melanogaster.

A typical URL to link to a gene report page: http://intermine.modencode.org/query/portal.do?externalid=FBgn0004053&class=Gene&origin=flybase

For a protein: http://intermine.modencode.org/query/portal.do?externalid=ZEN1_DROME&class=Protein

If you would like to link to a particular type of entry and have suggestions for additional templates you would like to see there please let us know, this is easy to configure.

Links to List Analysis pages

It is possible to link directly to a list analysis page by providing a list of identifiers in the URL. modMine will then search for the corresponding entries and create a modMine List with them. It is mandatory to specify the class of the data to load, i.e. Gene, Protein and it is important to respect the font case. For a list of data types please refer to the list of data types on the QueryBuilder page: http://intermine.modencode.org/query/customQuery.do

The URL to use takes the form: intermine.modencode.org/query/portal.do?externalids=identifier1,identifier2,identifier3&class=ClassName&origin=MySite

The origin in the URL is optional but will allow us to show a specific welcome/help message to incoming users from your site. It also helps us keep track of incoming links. Let us know what you want to use.

As an option, you can specify an organism name by using organism in the URL. The organism name must be the shortName as defined in modMine, for instance D. melanogaster.

A typical URL to link to a gene report page: http://intermine.modencode.org/query/portal.do?externalids=CG2262,CG3069,CG2859,CG5041,FBgn0036513&class=Gene&origin=flybase

Converting data types

When linking to a report page or a list analysis page you can convert the data type, for instance if you provide a Protein identifier and want to link to the corresponding Gene, you need to specify the class as Gene.

The URL to use takes the form: intermine.modencode.org/query/portal.do?externalid=proteinId&class=Gene

Examples:

  • http://intermine.modencode.org/query/portal.do?externalid=EVE_DROME&class=Gene
  • http://intermine.modencode.org/query/portal.do?externalids=CG2262,CG3069,CG2859,CG5041,FBgn0036513&class=Protein

Links to data categories

Links directly to data category pages in the latest release take the form:

intermine.modencode.org/query/aspect.do?name=aspect+name

Where aspect+name is the name of the aspect as shown on the modMine home page - match case and replace any spaces with '+'.

For example:

  • http://intermine.modencode.org/query/aspect.do?name=Genomics
  • http://intermine.modencode.org/query/aspect.do?name=Gene+Ontology
